# 🎵 Sketch Samples

Веб-приложение для музыкантов — позволяет исследовать музыкальные инструменты, воспроизводить их звучание и вести личный дневник музыкальных идей.

---

## Стек технологий

- **Angular 18** — standalone-компоненты, Reactive Forms, Angular Router
- **TypeScript 5.4**
- **SCSS** — кастомные CSS-переменные для единой темы
- **json-server** — REST API на основе `db.json` (порт 3000)
- **RxJS** — реактивное управление состоянием через `BehaviorSubject`
- **concurrently** — одновременный запуск фронтенда и бэкенда

---

## Возможности

- **Галерея инструментов** — 15 карточек музыкальных инструментов с изображениями
- **Интерактивное пианино** — кликабельные клавиши в боковой панели воспроизводят звуки выбранного инструмента (7 белых + 5 чёрных клавиш, ноты C–B с полутонами)
- **Избранное** — добавляйте и удаляйте инструменты в избранное; данные сохраняются на сервере
- **Дневник идей** — создавайте, редактируйте и удаляйте текстовые записи (название, дата, описание)
- **Аутентификация** — вход и регистрация с валидацией форм; сессия хранится в `localStorage`
- **Профиль пользователя** — просмотр данных, удаление аккаунта, выход из системы

## Структура проекта

```
src/app/
├── common-ui/
│   ├── layout/          # Боковая панель: навигация, пианино, превью инструмента
│   ├── music-card/      # Страница дневника идей (контейнер)
│   ├── add-posts/       # Форма создания записи
│   ├── edit-posts/      # Форма редактирования записи
│   ├── all-posts/       # Таблица всех записей пользователя
│   └── footer/          # Футер
├── pages/
│   ├── main/            # Галерея инструментов
│   ├── profile/         # Профиль и избранное
│   ├── login/           # Страница входа
│   ├── registration/    # Страница регистрации
│   └── four-hundred-four/ # Страница 404
└── data/
    ├── models/          # Post, User, Message
    ├── services/        # PostService, UserService, UserInfoService, MusikService
    └── Dbs/db.json      # База данных json-server
```

## Маршруты

| Путь | Компонент | Описание |
|---|---|---|
| `/` | → redirect | Перенаправляет на `/system/main` |
| `/system/main` | `MainComponent` | Галерея инструментов |
| `/system/profile` | `ProfileComponent` | Профиль и избранное |
| `/system/music-card` | `MusicCardComponent` | Дневник идей |
| `/login` | `LoginComponent` | Вход |
| `/registration` | `RegistrationComponent` | Регистрация |
| `/**` | `FourHundredFourComponent` | 404 |

## Установка и запуск

### Требования

- Node.js v18+
- npm v9+

### Установка зависимостей

```bash
npm install
```

### Запуск в режиме разработки

Запускает Angular dev-server и json-server одновременно:

```bash
npm start
```

- Приложение: [http://localhost:4200](http://localhost:4200)
- API: [http://localhost:3000](http://localhost:3000)

## API (json-server)

Сервер поднимается на основе файла `src/app/data/Dbs/db.json`.

| Метод | Endpoint | Описание |
|---|---|---|
| `GET` | `/users?email=...` | Найти пользователя по email |
| `POST` | `/users` | Создать пользователя |
| `PUT` | `/users/:id` | Обновить данные пользователя (в т.ч. избранное) |
| `DELETE` | `/users/:id` | Удалить пользователя |
| `GET` | `/record` | Получить все записи |
| `POST` | `/record` | Создать запись |
| `PUT` | `/record/:id` | Обновить запись |
| `DELETE` | `/record/:id` | Удалить запись |

## Заметки о реализации

- Защита маршрутов реализована вручную в `LayoutComponent`: при отсутствии `user` в `localStorage` происходит редирект на `/login`
- Состояние выбранного инструмента передаётся между компонентами через `MusikService` (`BehaviorSubject`)
- Звуки инструментов хранятся в `public/assets/mus/{index}/{note}.mp3`
- Изображения инструментов — в `public/assets/img-for-instruments/{index}.jpg`
- Пароли хранятся в `db.json` открытым текстом — проект учебный, не предназначен для продакшна
